L’objectif de la SAE15 est de prouver que vous êtes capable de récupérer des données sur un réseau, de les transformer dans un format cible après avoir réalisé des calculs et de les transmettre sur un serveur web. Et ceci doit se faire de façon automatisée !

Présentation du projet
Vous devez réaliser un « script » capable de récupérer des données, au format CSV, sur le site https://www.data.gouv.fr/. Les données seront ensuite agrégées (calcul de moyennes, maximum, filtrage…), puis transformées au format JSON afin de les transmettre à un site web qui vous les affichera au format graphique. Le choix des données à traiter est à votre discrétion ainsi que les traitements à faire.

Pour réaliser ce projet, vous devrez utiliser GitLab pour stocker vos scripts, code et document à rendre.

Déroulement
Pour la réalisation du projet, vous aurez à votre disposition 2 h de cours et 22 h de TP dont 2 pour l’évaluation et dont 16 h en autonomie. L’enchaînement des séances se fera selon le tableau ci-dessous.

Les étapes du projet
Vous devrez donc dans un premier temps constituer une équipe de travail de 2 ou 3 étudiants tout en choississant le sujet sur lequel vous allez travailler. ATTENTION, certains sujets ne se font qu'à 2.

Ensuite, l’un de vous devra initialisera le projet dans le GitLab de l’Artois et invitera les autres membres de son équipe ainsi que l’enseignant référent. Tous les documents à rendre du projet seront stockés dans le gitLab.

Puis, vous  plannifiez les tâches à faire après les avoir identifiées ensemble. Vous pouvez vous appuyer sur les compétences à mettre en œuvre lors de la SAE !

Après vous êtes réparti équitablement le travail, vous pourrez démarrer le codage du script. Le choix des technologies est à votre entière discrétion. Néanmoins, ce choix devra être expliqué lors de l’évaluation. Il ne faut pas perdre de vue que le but est de réaliser un fichier de données JSON qui sera transféré à un site web afin d'être mis sous forme graphique. Ce site est: https://quickchart.io  (voir un exemple en Python Code Python

À chaque fois que vous travaillerez sur le projet, il faudra le mettre à jour dans le GitLab. Les enseignants peuvent à tout moment regarder votre travail et poser des questions/remarques dans le GitLab.

Modalités d’évaluation[Modifier]
Vous serez évalué sur votre organisation. Chaque membre de l'équipe doit travailler. N'oubliez pas de faire une page d'acceuil (fichier readme.md) dans le gitlab afin de guider les utilisateur qui viennent voir votre projet.

GitLab
L’évaluation des scripts se fera par le GitLab. C’est le seul et unique endroit où doit se trouver vos fichiers. La fréqunetation de chaque étudiant sur le gitlab sera évaluée ainsi que l'implication de chacun. C'est pour cela qu'il faut que chacun travail avec SON login sur le GitLab !

Portfolio
Vous devez remplir votre portfolio au fur et à mesure de l’avancement du projet. Celui-ci doit également se trouver dans le GitLab (répertoire Portfolio, un fichier nominatif par étudiant)

Présentation orale et questions à l’orale
Une présentation orale de 10 minutes de votre projet sera planifiée lors de la dernière séance de la SAE. Vous ferez une démonstration de l’exécution de votre script et vous expliquerez comment celui-ci fonctionne. À l’issue de la présentation, un temps de questionnement individuel sera réalisé. Les documents utilisés pour la présentation, les portfolios individuel seront téléchargés de votre gitlab au début de la séance d'évaluation.
